What finally clicked today?
-
- Git is a version control tool that tracks changes made to a project over time.
- Every commit creates a snapshot of my progress, allowing me to go back if something breaks.
- Git doesn't automatically save my work. I have to stage my changes and commit them.

What confused me?
-
- Why was Git important to a developer?
- How does VS Code, Github, and Git work together?
- How does the Git workflow actually work?

What would I explain differently now?
-
- Git is a version control system that tracks the history of a project, not just a tool for saving files.
- VS Code is where I write and edit code.
- Git keeps track of all changes made to the project.

How does this relate to cybersecurity?
-
- Cybersecurity professionals such as Security Automation Engineers, Detection Engineers, SOC Analysts, and Software Developers often create scripts, detection rules, and automation tools. Git allows teams to track every change made to those projects, collaborate safely, and roll back to a previous version if a mistake is introduced. This helps maintain accurate documentation and ensures changes can be audited throughout the development process.
